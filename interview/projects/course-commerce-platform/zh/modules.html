05重点模块

5 个重点模块，逐一拆解

每个业务域独立成模块，但共享底层的路由、权限、请求、表单和组件。下面按门类逐一过一遍。

01 · 项目中心 template/

运营做活动的起点。"一个项目" = 一条落地页 + 它的所有版本。 下设四个子模块：

已有项目
课程项目 / 导航项目，项目列表 + 版本管理 + 复制 / 编辑 / 预览 / 上线 / 下线。
模板中心
课程落地页模板的集中管理。运营基于模板快速创建项目，大大降低重复配置成本。
组件库
5 种通用组件（弹窗 / 完成页 / 短信 / 表单 / 按钮素材）的独立增删改查，供落地页配置时复用。
小程序列表
加微小程序相关配置的列表化管理。
02 · 礼品中心 present/

管"拿来送"的一切东西。商品模型三大类：

课程商品
课程供应链（SCM）+ 商品中心。把离散的课程 SKU 包成可售卖、可赠送的商品对象，带价格、规格、库存、归属。
实物商品
课程盒子 / 实物盒子 —— 真实物流发货的商品，对接仓储和履约。
虚拟商品
优惠券 —— 创建、发放规则、有效期、使用限制、批次管理。
深层嵌套表单 批量操作 字段联动
03 · 小工具 tools/

运营日常最高频使用的模块。"小"是相对模块大小说的，对业务的价值很不小。

推广工具
活动推广链接与二维码的生成 / 管理 / 数据回看。
定制活动
面向非标活动场景的配置入口，把无法走标准流程的定制需求收敛进来。
渠道活码
渠道归因核心能力：一个二维码背后挂多个客服号，扫码自动分配 + 归因统计。
排查工具
给运营和客服用的自助排查面板。输入用户 ID，快速看到下单、优惠券、归因链路，省掉找研发的时间。
04 · 用户画像 user-portrait/

把"哪些用户能买哪些课"这件事规则化。本质上是一套规则引擎的前端配置界面。

用户限购
按用户标签、身份、消费历史等维度配置限购规则。
课程限购
按课程维度反过来配置：这门课谁能买、每人几份、有没有梯度。
课程关系
定义课程之间的互斥、替代、升级关系。买了 A 是否能买 B，买了高级版能否买入门版，都在这里配。
规则引擎前端 Formily 动态 Schema 条件组合器 UI
05 · 工作台（低代码搭建器）work-space/

运营自助生成课程落地页的入口。核心能力是7 步向导 + Formily 驱动的表单配置 + 手机端实时预览 + 保存 / 预览 / 上线闭环。 8 种落地页模板，支持多个投放承载（H5、小程序），带版本管理和草稿机制。

低代码页面搭建是团队的另一个亮点项目（Low-Code Campaign Platform），那边的深度和完整性更高。 这里只作为"整个商业后台中的一个模块"存在，故不再展开。如需了解完整的低代码方案，请移步 Low-Code Campaign Platform。

模块协同上的几个关键设计

  • 同一份权限模型覆盖所有模块 —— access.ts 里几十个权限 key 覆盖菜单、路由、按钮。新模块加一个 key 就接入全站。
  • 通用组件跨模块复用 —— 表单搜索组、列表操作下拉、上传、Formily 扩展表单项在所有模块里一致的样子和行为。
  • API 层类型自动生成 —— 后端改接口 → Swagger 拉一次 → 前端编译就能发现不兼容的地方。
  • 共享的 Redux 切片 —— config（全局配置）/ user（用户信息）/ checkTool（排查工具状态），跨页面共享状态不用层层透传。
