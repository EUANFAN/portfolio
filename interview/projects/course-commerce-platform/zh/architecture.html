关键链路 1：权限链路（菜单 + 按钮双层鉴权）

项目里最有辨识度的一条——后端权限列表直接驱动前端的路由、菜单、按钮三件事。

SSO 登录 → getInitialState() → 拉 {menus, permissions} → 按 type 分层 → access.ts 生成 boolean 开关 → 路由守卫 + 菜单渲染 + 按钮鉴权

权限列表里 type=0/1 是菜单权限（驱动路由和 ProLayout 菜单渲染）， 其余是功能权限（驱动按钮）。菜单层用 <Access accessible={access.canSeeXxx}>， 按钮层用 useAuthCheck('project-center-project-list-course-edit') —— 一处 key，路由跳转、菜单显隐、按钮可见三件事同时生效，新业务模块只需要加 route + access key，不用手动改菜单。

关键链路 2：请求与契约链路

前后端契约由 Swagger 自动生成，业务代码只写 happy path。

@dandelion/swagger → 生成 TS 类型 + 请求函数 → 业务调用 → 请求拦截器（SSO token / mode 注入） → 后端 → 响应拦截器（401 跳 SSO · 业务码统一 toast）

@dandelion/swagger 基于 OpenAPI 自动生成 TS 类型和请求函数，前端几乎不再手写接口层； axios 拦截器把 401 自动登出并跳 SSO、业务错误码统一弹 toast 两件事做掉， 错误约定在拦截器里收敛，不扩散到业务层——业务代码里只需要关心 happy path。

关键链路 3：CRUD 列表页链路（Hook 驱动）

20+ 列表页共享同一套行为，单页代码从 500+ 行压到 ~150 行的核心抽象。

columns + searchForm + fetchFn 声明 → useTableSearch → useAntdTable 数据源 → useTableSet → ProTable 渲染 + localStorage 持久化

useTableSearch 接管搜索 / 重置 / 分页 / URL 同步——返回 { search, reset, formValue }， 搜索时自动 reset current，翻页时保留筛选，重置时清 params； useTableSet 接管自定义列（列宽 / 列显隐 / 操作列），返回 { width, render }， 用户调整的列配置走 localStorage 持久化。 默认行为在 Hook 里集中定义，20+ 列表页开箱一致，新增一页不用再想一次。

关键链路 4：资源上传链路（前端直传 OSS）

图片 / 视频 / 文件统一走 OSS 签名直传，不经过业务服务器中转。

选文件 → 前端校验（类型 / 大小 / 尺寸） → getOssToken() → FormData POST 到 OSS host → CDN_NAME/PUBLIC_PATH/fileUrl → 写回表单字段

统一的 <Upload> / <VideoUpload> / <Dragger> 组件把链路封装好： 调 /libraapi/oss/get_policy_token 拿 { OSSAccessKeyId, signature, policy, host, dir }， 然后用 FormData 把 key / OSSAccessKeyId / policy / signature 连同文件一起 POST 到 OSS host， 回拿到 fileUrl 后拼成 OSS_CDN_NAME/OSS_PUBLIC_PATH/fileUrl 入表单。 前端直传、业务服务器零转发，减轻后端压力，也避免大文件走业务链路超时。 视频播放用 ?x-oss-process=video/snapshot 直接生成封面，无需后端再算一次。
