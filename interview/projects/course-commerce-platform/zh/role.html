03我的职责

我在这个项目里做了什么

我在这个项目担任 前端负责人，负责整体前端架构设计 + 核心模块开发 —— 把 Umi Max 的底座搭起来，把权限、请求、表单、组件这些公共能力抽成一套工程级规范， 再自己下场把几个重点业务页做完、做稳。

整体前端架构

  • 前端底座的搭建 —— 基于 Umi Max + React + TypeScript 的工程脚手架、约定式路由配置、主题与布局、多环境构建（dev / mock / gray / prod）。
  • 细粒度权限体系的落地 —— 基于 Umi Max 的 access 插件 + initialState，把菜单权限和功能权限拉齐，做路由级和按钮级控制，配合 403 页兜底。
  • 统一的请求与鉴权层 —— axios 拦截器集成 SSO token、统一错误提示、401 自动登出并跳 SSO；用 @dandelion/swagger 自动生成 API 类型，前端几乎不再手写接口层。
  • 工程规范与协作约定 —— ESLint / Prettier / Stylelint / Husky / lint-staged / commitizen 一套落地；目录分层（业务模块 / 平台能力 / 基础设施）和命名规范也在这一步定下来。

核心模块开发

  • 通用组件库 —— 表单搜索组、列表操作下拉、上传、筛选、Formily 扩展表单项（级联 / 多选 / 数字 / 开关 / trim 输入等）、排序拖拽、机型预览器等 ~20 个通用组件。
  • CRUD 场景抽象 —— 沉淀 useTableSearch / useTableSet 等 Hook，把筛选 / 分页 / 排序 / 列持久化 / URL 同步一处接管，20+ 列表页从 500+ 行压到 ~150 行。
  • 核心业务页面的落地 —— 项目列表、模板中心、商品中心、优惠券、限购规则、推广工具、渠道活码等多个业务页面的前端实现与上线。
  • 工作台（页面搭建器） —— 作为一个业务模块参与了核心流程（7 步向导、Formily 表单、手机端实时预览、保存 / 预览 / 发布闭环）。因为低代码不是本项目的亮点，此处仅列出。

简单讲：架构我拍板，核心模块我自己写。 小团队、多业务线，定位是"既搭底座、也上战场"—— 既要把底座搭好、把规范定住，也要和兄弟们一起扛业务模块的实际交付。
