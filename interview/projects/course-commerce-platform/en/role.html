03Role & Ownership

What I actually did on this project

I was the frontend lead on this project, responsible for the overall frontend architecture and the delivery of the core modules. In practice, that meant setting up the Umi Max foundation, pulling permissions, requests, forms, and shared components into a real engineering model, and then personally taking several key business pages all the way to production.

Overall frontend architecture

  • Built the frontend foundation — the project scaffold on top of Umi Max + React + TypeScript, centralized route config, theme and layout, and multi-environment builds for dev / mock / gray / prod.
  • Implemented the fine-grained permission model — based on Umi Max access plus initialState, we aligned menu permissions and function permissions, and applied them at both route level and button level, with a 403 fallback page.
  • Built the unified request and auth layer — axios interceptors handled SSO token injection, global error messaging, and automatic logout plus SSO redirect on 401. @dandelion/swagger generated API types, so the frontend barely had to handwrite the API layer.
  • Set the engineering conventions — ESLint, Prettier, Stylelint, Husky, lint-staged, and commitizen were all put in place, and we also fixed the directory layering and naming conventions at this stage.

Core module delivery

  • Shared component layer — search groups, table action dropdowns, upload components, filters, Formily field extensions such as cascaders, multi-selects, numeric fields, switches, trim inputs, sortable drag-and-drop, and the device previewer. Roughly 20 shared components in total.
  • CRUD abstraction — hooks like useTableSearch and useTableSet, which centralized filtering, paging, sorting, column persistence, and URL sync. More than 20 list pages went from 500+ lines down to about 150 lines.
  • Key business pages — I directly worked on the frontend delivery and launch of project lists, template center, product center, coupons, purchase-limit rules, promotion tools, and channel QR code flows.
  • Workspace / page builder — I also participated in the core flow of that module, including the 7-step wizard, Formily forms, live mobile preview, and the save / preview / publish loop. It is listed here as one business module, but low-code itself is not the main highlight of this project.

In short: I set the architecture direction, and I also wrote the key modules myself. It was a small team, spread across several business domains, so the role was very much “build the foundation and still go fight the actual business battles.”
