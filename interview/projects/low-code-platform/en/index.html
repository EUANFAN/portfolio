01Overview

What it is, in one sentence

A low-code platform where non-engineers compose marketing campaign pages by dragging schema-driven widgets, preview them live on iOS/Android frames, and publish to multiple channels (mobile web, WeChat H5, WeChat Mini Program) without any engineering involvement.

Situation
Every campaign page was hand-built by a frontend engineer. Ops had a steady queue of "change this image, move that button, ship by tomorrow" requests; engineers spent a disproportionate amount of time on repetitive layout work.
Task
Take engineering out of the campaign delivery loop without losing brand and visual quality, and support multiple channels from a single source of truth.
Action
Designed a JSON schema that fully describes a page and its interactions; built a rendering engine, a versioned widget library loaded from CDN, and a drag-and-drop editor with live multi-device preview. Drove adoption across 4 business lines.
Result
Campaign launch time dropped from ~2 days to ~15 minutes. Frontend team shifted from delivery work to platform work — widget library, editor UX, analytics integration.

Key ideas that made it work

  • Schema is the contract. One JSON describes layout, widgets, styles, triggers and share config. Editor writes it; runtime reads it. Nothing else crosses the line.
  • Widgets are versioned and CDN-loaded. Each widget is a self-contained bundle with a semver; the runtime picks the version pinned in the schema, so old pages stay stable when the library evolves.
  • Triggers over code. Instead of letting ops write JS, we exposed a small set of declarative triggers (onClick, onShow, onScroll) whose actions are also schema — navigation, tracking, share, modal, etc.
  • Cross-channel by construction. The same schema renders to mobile web, WeChat H5 and Mini Program via channel-specific adapters; ops configures once, publishes everywhere.
